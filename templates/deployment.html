Section 03 — API

Prediction Endpoint

A single REST endpoint accepts the 9 welfare indicators as JSON and returns the Random Forest risk classification with class probabilities. No authentication required.

POST /api/predict — Predict abandonment risk from zone indicators

Request body (JSON)

// All numeric — 9 feature variables { "citizen_reports": 45, "abuse_cases": 12, "animals_seized": 8, "vet_emergencies": 20, "microchip_coverage": 0.25, "sterilization_rate": 0.40, "stray_density": 85.5, "prior_cases_6m": 30, "institutional_response_days": 7 }

Response (JSON)

{ "ok": true, "rf": { "label": "HIGH", "proba": { "HIGH": 82.4, "MEDIUM": 14.1, "LOW": 3.5 } } }

Section 04 — Render.com

Deployment on Render

PetWatch is deployed as a Web Service on Render's free tier. The platform auto-detects Python and serves the Flask application via gunicorn. Every push to main triggers a new build.

01
GitHub Repository
Source code is pushed to a public GitHub repo. Render connects directly via OAuth — no manual configuration needed.
02
Build Command
pip install -r requirements.txt installs Flask, scikit-learn, pandas, numpy, and gunicorn automatically.
03
Start Command
gunicorn app:app launches the production server on Render's assigned PORT environment variable.
04
Auto-Deploy
Every push to main triggers a new build. Zero-downtime deploys are handled by Render automatically — no manual intervention required.
05
Dataset & Model
The synthetic CSV is bundled in the repository. The RF model is retrained in-memory on each cold start — no serialized model file required.
