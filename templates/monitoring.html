Section 03 — Strategy

Monitoring Framework

Six monitoring pillars covering performance, drift, operations, concept shift, audit logging, and stakeholder reporting — each with a defined ownership and cadence.

[P1]
Performance Monitoring
Track accuracy, F1-score, and per-class precision on incoming batches monthly against a held-out validation set.
Automated
[P2]
Data Drift Detection
KS-tests compare incoming feature distributions against the training baseline. Alert when shift exceeds 15% on key variables.
Alert triggered
[P3]
Operational Health
Monitor API response times, request volume, error rates on /api/predict, and Render uptime. Alert above 2% error rate.
Manual review
[P4]
Concept Drift Tracking
Validate predictions against new ground-truth labels from IDPYBA institutional reports when available.
Manual review
[P5]
Audit Logging
Every prediction request is logged with inputs, predicted label, probabilities, and timestamp for retrospective analysis.
Automated
[P6]
Stakeholder Reporting

Quarterly

Section 04 — Retraining

Retraining Protocol

A five-step procedure is triggered when CV accuracy drops below 90%, when KS-test p-value < 0.05 for two or more key variables, or on a fixed 6-month schedule.

01
Trigger Detection
Retraining fires when CV accuracy < 90%, or when feature distribution KS-test p-value < 0.05 for 2+ key variables, or on the 6-month fixed schedule.
02
Data Collection & Augmentation
Collect new observations from IDPYBA or municipal sources. Augment the synthetic dataset with verified records, maintaining 6,500+ total records.
03
Model Retraining
Retrain RandomForestClassifier (n_estimators=200, max_depth=8). Run 5-fold CV. Only promote if new CV accuracy ≥ 90% and exceeds the current model's score.
04
Shadow Mode Validation
Run new model in parallel for 2 weeks. Compare predictions against the live model on real traffic. Promote to production only if agreement rate ≥ 85% on stable cases.
05
Deploy & Document
Push updated code to GitHub. Render auto-deploys. Update the model card with new metrics, training date, dataset version, and architecture changes.

Section 06 — Long-Term Risks

Future Risks & Mitigation

Even a well-monitored system faces long-term challenges as the world it models continues to change.

[POLICY]
Regulatory Changes
New city regulations on microchipping or sterilization subsidies could shift feature distributions rapidly, invalidating trained correlations.
[DATA]
Data Availability
Dependency on IDPYBA data means format changes or access restrictions could interrupt the data collection and retraining pipeline.
[DRIFT]
Model Staleness
Without real-world labels, the model continues evaluating against synthetic ground truth. True production accuracy remains uncertain until validated against real outcomes.
[GEO]
Coverage Gaps
Sumapaz and rural localities have very few records. Predictions for underrepresented zones are less reliable and could skew resource allocation.
[INFRA]
Infrastructure Limits
Render's free tier cold-starts after inactivity (30–60 seconds). High-traffic periods may require migrating to a paid tier or a more robust host.
[PRIVACY]
Data Privacy
If extended to citizen-level data, Colombian Law 1581 (GDPR-equivalent) protections must be implemented before any production expansion.
